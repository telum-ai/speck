# Coverage Matrix — [PROJECT_ID | EPIC_ID]

The runtime analog of `traceability-matrix.md`: every reachable **cell** either RESOLVES (RUN with real evidence, DEFERRED-BY-BUDGET with a reason, or N-A with a reason) or is a **GAP** — an in-universe, un-run, un-waived cell, which is itself a P3 finding ("can't reach it is a finding"). Breadth gaps are **visible, not silent**. Generated by `generate-coverage-matrix.sh`; gated by `validate-coverage-matrix.sh`. The breadth verdict only **caps**, never raises, the claimable readiness state.

## Coverage universe (dimensions)

A cell = one tuple of:
- **persona** — evidence-contract §4 named personas + the archetype persona-army (naive-hostile, connoisseur-hostile, returning-user, skeptical-buyer, power-user, a11y-first [keyboard+SR], locale-switcher, low-bandwidth, impatient).
- **route/screen** — mechanically enumerated (web: experience-chain §2 chain / router manifest). `infra_service`/`backend_api`: endpoints.
- **state** — {happy, error, empty, loading}. `infra`: the §11 disruption class {nominal, concurrency, DB-drop, rate-limit, AI-down, cross-user}.
- **viewport** — recipe `visual_testing.breakpoints` (mobile / tablet / desktop / …).
- **theme** — {light, dark}.
- **input-sample** — N samples (default 5) **only** on §8 AI-generative surfaces.

## Sampling policy (budget)

- **Critical-core** (product-contract §5 magic-moment surfaces + the flagship): **full cross-product**.
- **Secondary** surfaces: **one representative cell per dimension-value** (every value appears ≥1) — declare below what was reduced. `torture` tier expands secondary toward full; the reduction is disclosed, never silent.
